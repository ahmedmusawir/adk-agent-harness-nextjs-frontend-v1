### Test wisdom captured (informal)

For the Copy action test, fighting jest + jsdom + clipboard API mocking was a 4-iteration rabbit hole — the mock wasn't being called even though the handler clearly ran (button transitioned to "Copied" state). Switched to **behavior-based assertion**: assert the button's aria-label flips to "Copied" after click. Tests user-observable behavior directly without mocking browser APIs. More robust, less brittle, and aligns with Rule K9 (test intent, not implementation).

Generalizable rule for future test-writing: when mocking a browser API in jest is fighting you, switch to asserting the user-observable state change. The state change is the actual behavior we care about — the API call is an implementation detail. Worth promoting to Lesson 8 at Phase 8 retrospective if the pattern reappears.

## Morning hotfix (2026-05-29 ~11:30 AM)

Operator ran `npm run dev` and `npm run build`. Dev worked (with a flake on first run); build failed:

```
./src/utils/supabase/server.ts:3:1
You're importing a module that depends on "next/headers". This API is only
available in Server Components in the App Router, but you are using it in
the Pages Router.
Client Component Browser:
  ./src/utils/supabase/server.ts
  ./src/utils/get-user-role.ts
  ./src/components/layout/CyberizeSidebar.tsx
```

**Diagnosis:** In Phase 5.5 I added `import { AppRole } from "@/utils/get-user-role"` to `CyberizeSidebar.tsx` (a `"use client"` component). The whole `get-user-role.ts` module bundles for the client when imported as a value, dragging `next/headers` via the file's `getUserRole` server function. Build correctly rejected.

**Fix (5 min):**
- Created `src/utils/app-role.ts` containing only the `AppRole` enum (no server-only imports)
- Updated `src/utils/get-user-role.ts` to import + re-export AppRole from `app-role.ts` (backward compat)
- Updated `src/components/layout/CyberizeSidebar.tsx` to import `AppRole` from `@/utils/app-role` directly

**Verified:**
- `npm run build` → clean, 21/21 static pages generated, all routes present including `/chat` and `/mission-control`
- `npm test` → 18 suites / 115 tests pass

**Captured as Lesson 8 in `STARTER_KIT_FEEDBACK.md`:** "Don't import server-only modules into client components, even transitively." Plus memory `feedback_client_bundle_server_only_imports.md` for cross-session recall. Kit-level fix opportunity noted: ship `AppRole` already extracted in the pristine starter.
