## Phase 5.4 — Page Composition Correction + AppShellPage primitive

Operator review of Phase 5 caught a kit-convention violation: `ChatPageContent.tsx` was authored in `src/components/chat/` instead of co-located with `src/app/(cyberize)/chat/page.tsx`. Also no usage of the kit's common primitives (Page/Row/Box). UI-UX-BUILDING-MANUAL.md §"Page Building Pattern" (line 412+) documents exactly the convention I missed; `(public)/demo/` is the canonical example.

### What was done

- **File moves**: 4 chat files moved from `src/components/chat/` to `src/app/(cyberize)/chat/` (co-located with `page.tsx`):
  - `ChatPageContent.tsx`, `MessageBubble.tsx`, `MessageList.tsx`, `ChatInput.tsx`
  - `AgentSwitcher.tsx` stays at `src/components/chat/` (legitimately shared between sidebar + chat content)
- **New common primitive `src/components/common/AppShellPage.tsx`** — full-bleed app surface with sidebar + main, mobile-responsive built-in (hamburger-triggered slide-over drawer at `<md`, persistent sidebar at `≥md`). Heavy in-file JSDoc covering: when to use, decision tree vs Page+Row+Box, mobile behavior contract, theme conventions, accessibility notes, intentional omissions, promotion path.
- **`(cyberize)/layout.tsx`** refactored to use `<AppShellPage sidebar={<CyberizeSidebar />} mobileTitle="CYBERIZE" mobileTopBarRight={<ThemeToggle />}>`. Dark bg moved from `zinc-950` to `zinc-800` (softer dark per operator feedback).
- **`(cyberize)/chat/page.tsx`** updated to use relative import `./ChatPageContent`.
- **Test imports** updated for the new co-located paths (3 test files).
